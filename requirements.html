EXIF Annotator - Business Requirements

1. Core Purpose

Create a Python-based application that processes digital photographs by adding EXIF metadata directly onto the image in a visually appealing border format.

2. Image Processing Requirements

2.1 Input Processing

  • Accept JPG image files as input
  • Read and extract EXIF metadata from input images
  • Handle cases where EXIF data is missing or incomplete
  • Support Unicode filenames

2.2 Image Modification

  • Add a black border to the bottom of the image
  • Border height: 110 pixels
  • Preserve original image quality and dimensions
  • Maintain original image metadata in the output file

3. EXIF Data Display Requirements

3.1 Text Format

  • Display EXIF data in two lines within the border
  • Use Arial font at 32pt size
  • White text on black background for maximum readability

3.2 Data Layout

Line 1 (at 30% of border height): Date/Time, Camera Model, Lens Model
Line 2 (at 70% of border height): Focal Length, F-Stop, Shutter Speed, ISO

3.3 Missing Data Handling

  • Display "N/A" for any missing EXIF fields
  • Maintain consistent formatting regardless of missing data

4. Technical Requirements

4.1 Development Standards

  • Implement in Python using Pillow for image processing
  • Use exif library for metadata extraction
  • Follow PEP 8 coding standards
  • Implement comprehensive pytest test suite

4.2 Project Structure

  • Maintain clear project organization
  • Include proper documentation (README.md)
  • Provide test coverage for all major functions
  • Include sample images for testing

5. Performance Requirements

  • Process individual images efficiently
  • Support batch processing of multiple images
  • Maintain original image quality
  • Handle various image sizes and resolutions

6. Documentation Requirements

  • Provide clear installation instructions
  • Include usage examples and documentation
  • Document all dependencies and versions
  • Maintain testing documentation
